## Recommended Execution Order
1. Establish Workspace Validation And UI Foundation - 0-5%; lock the validation baseline and shadcn/Midday UI foundation before product work.
2. Define Brain Core Contracts - 5-10%; shared JSON types unblock Rust readers, client wrappers, UI, dispatch, and scheduler work.
3. Implement Rust Brain State Readers - 10-18%; read-only state is the safest first native capability.
4. Implement Atomic Brain JSON Writes And Locks - 18-25%; all later mutations depend on safe writes.
5. Build Midday/Shadcn Desktop Shell - 25-32%; gives the app a durable Midday-style route/component structure.
6. Build Project Configuration Surface - 32-40%; enables and validates project registry management.
7. Build Queue Dashboard And Filters - 40-50%; exposes queue state and makes the app useful for daily monitoring.
8. Implement Manual Implementation/Review Dispatch - 50-60%; adds controlled one-shot runner execution.
9. Add Auditable Run Logs And Transcripts - 60-68%; makes every dispatch traceable.
10. Add PTY-Backed Thread Terminals - 68-76%; turns one-shot runs into interactive threaded sessions.
11. Add Approval Broker And Cards - 76-84%; preserves user control over sensitive runner actions.
12. Add Background Scheduler Controls - 84-90%; automates implementation/review ticks within project limits.
13. Add LaunchAgent Helper Support - 90-96%; enables durable background operation.
14. Add Notifications, Packaging, And Release Readiness - 96-100%; finishes desktop ergonomics and release verification.
